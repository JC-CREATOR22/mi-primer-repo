Glosario de términos e instrucciones - Práctica 1

Términos e instrucciones utilizadas:
- git --version: Comando para verificar si Git está instalado y mostrar su versión.
- git config --global user.name "Tu Nombre": Configura el nombre de usuario global para commits.
- git config --global user.email "tucorreo@ejemplo.com": Configura el correo electrónico global (usar el de GitHub).
- git config --list: Muestra la configuración actual de Git.
- mkdir mi-primer-repo: Crea una nueva carpeta llamada mi-primer-repo.
- cd mi-primer-repo: Cambia el directorio actual a mi-primer-repo.
- git init: Inicializa un nuevo repositorio Git en la carpeta actual.
- echo "# Mi primer repositorio" > README.md: Crea un archivo README.md con el texto especificado.
- git status: Muestra el estado actual del repositorio (archivos modificados, no rastreados, etc.).
- git add README.md: Añade el archivo README.md al área de preparación.
- git commit -m "Primer commit: añadir README": Guarda los cambios en un commit con un mensaje descriptivo.
- git remote add origin https://github.com/JC-CREATOR22/mi-primer-repo.git: Conecta el repositorio local con el remoto en GitHub.
- git remote -v: Verifica los remotos configurados y sus URLs.
- git branch -M main: Renombra la rama principal de master a main.
- git push -u origin main: Sube los cambios locales a la rama main del remoto origin.
- git clone https://github.com/usuario/repositorio.git: Copia un repositorio existente de GitHub a tu computadora local.
